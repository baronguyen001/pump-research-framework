# Strategy Reframe

A single 100-point score is useful for communication, but different edges need
different playbooks. Treat these as separate research modes.

| Strategy | Edge | Detection with public tools | Expected profile | Backtestability |
|---|---|---|---|---|
| A. Pre-listing CEX play | Listing announcements can reprice liquidity quickly | Exchange announcement pages, project liquidity, venue history | Higher hit rate, lower upside, short holding period | High: announcements are public |
| B. Smart-wallet copy | Some wallets show repeat skill or information advantage | Cielo, Arkham, Nansen labels, public explorers | Lower hit rate, larger winners, cluster risk | Medium to high: wallet history is public |
| C. Narrative rotation | Categories trend before individual laggards reprice | CoinGecko categories, DefiLlama, Kaito, Token Terminal | Moderate hit rate, portfolio approach | Medium: category history exists but can be messy |
| D. Airdrop farming | Cost can be mostly time and gas before TGE | protocol usage, points pages, revenue, TGE docs | High variance, non-trade return profile | Low to medium: logs matter |
| E. Memecoin micro-bet | Lottery basket with strict loss budget | DexScreener, holder checks, liquidity, organic social | Very low hit rate, convex upside | Low: survivorship bias is severe |

## How to use this table

Do not apply the same position size, holding period, or exit rule across all
strategies. A memecoin basket and a category rotation basket should not share
the same base-rate assumptions.

The safest upgrade from the base framework is to record each strategy separately:
entry reason, max size, stop condition, outcome, and what evidence actually
predicted the result.

Trawlkit can automate collection, but it cannot remove the need for strategy
separation or out-of-sample validation.
