# Layer 4: Catalyst

Purpose: time the research. A good narrative can move sideways for months if no
event forces attention.

| Public tool | Use |
|---|---|
| CoinMarketCal | community event calendar |
| Token Unlocks | unlock cliffs and emissions |
| Binance announcements | tier-1 listing notices |
| Coinbase announcements | listings and roadmap updates |
| Upbit notices | Korean exchange listings |
| Project blogs | mainnet, TGE, partnership, and governance dates |
| Polymarket | public event probabilities |

Strong signals worth 20 or more points:

- A clear event is inside 30 to 90 days.
- Utility unlocks after mainnet, staking, fee share, or product release.
- A tier-1 listing rumor has plausible eligibility and liquidity context.
- A large unlock has already passed and sell pressure is fading.

Weak or red-flag signals:

- A major unlock or emissions cliff arrives inside 30 days.
- The team repeatedly sells the news.
- The event is vague: "soon," "big partnership," or "major update."
- The catalyst is already priced by mainstream attention.

How to wire this layer to your own data: maintain a dated event calendar with
source links, unlock size, and whether the event creates real demand or only a
temporary headline.
